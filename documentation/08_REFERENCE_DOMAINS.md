\# 08 — Reference Domains



\## Purpose



This document defines representative application domains for the Constitutional Game Runtime.



These domains demonstrate that the runtime governs constitutional execution rather than any specific game genre.



The runtime architecture is domain-general.



\## Domain Principle



The Constitutional Game Runtime governs evolving worlds.



A "world" may represent any persistent environment whose state changes over time and whose activity must remain constitutionally admissible.



\## Open-World Simulation



Examples:



\- Persistent exploration

\- Dynamic ecosystems

\- Survival worlds

\- Procedural civilizations



Governance continuously evaluates player and simulation activity against the current world revision.



\## Role-Playing Games



Examples:



\- Character progression

\- Quest systems

\- Reputation

\- Inventory

\- Factions



Actions remain admissible only while constitutional conditions remain satisfied.



\## Strategy Games



Examples:



\- Resource management

\- Territory control

\- Logistics

\- Diplomacy

\- Military operations



Governance evaluates strategic actions against continuously evolving world conditions.



\## Sports Simulation



Examples:



\- Rule enforcement

\- Tournament management

\- Player eligibility

\- Match governance

\- League operations



Simulation evolves independently of any individual participant.



\## Persistent Civilizations



Examples:



\- Population growth

\- Economic systems

\- Environmental evolution

\- Government

\- Infrastructure

\- Migration



The world continues evolving even when no player interaction occurs.



\## Autonomous Agent Systems



Examples:



\- NPC societies

\- AI factions

\- Robotics

\- Cooperative agents

\- Competitive agents



Every autonomous action remains subject to constitutional governance.



\## Educational Simulation



Examples:



\- Historical simulation

\- Scientific environments

\- Ecological systems

\- Economic education

\- Policy experimentation



Simulation emphasizes reproducible constitutional behavior.



\## Research Simulation



Examples:



\- Complex systems

\- Counterfactual experimentation

\- Long-duration simulations

\- Emergent behavior

\- Multi-agent research



Replayability supports scientific reproducibility.



\## Constitutional Generality



The runtime is not specialized for one genre.



Every domain shares the same constitutional primitives:



\- World

\- Governance

\- Execution

\- Simulation

\- Evidence



Domain-specific behavior emerges through world definition rather than changes to the constitutional execution model.



\## Summary



The Constitutional Game Runtime provides a domain-independent constitutional execution architecture.



Different applications define different worlds.



The constitutional runtime governing those worlds remains the same.
